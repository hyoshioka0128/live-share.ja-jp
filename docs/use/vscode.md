## <a name="share-a-project"></a>プロジェクトを共有します。

をダウンロードして Visual Studio Live Share をインストールしたら、コラボレーション セッションを開始し、お客様と協力する仕事仲間を招待する次の手順に従います。

1. **サインイン**

    Live Share の拡張機能をインストール、再読み込み、および依存関係のインストールが完了を待って後、は、2 人の他の共同作業者にサインインします。 参照してください[サインイン](#sign-in)の詳細。

2. **フォルダーを開く**

    通常、ワークフローを使用して、フォルダー、プロジェクト、またはゲストと共有するソリューションを開きます。

3. **[省略可能]非表示または除外されたファイルを更新します。**

    既定で Live Share**を非表示に**ゲストからの共有フォルダーの .gitignore ファイルで参照されているファイル/フォルダー。 **非表示**ファイルは、ゲストのファイル ツリーに表示されないようにします。 **除く**ファイルが、定義へ移動 と同様に、またはデバッグまたは「準拠」の中に、ファイルにステップ インする場合の状況でゲストを開いてから Live Share を妨げるより厳密なルールを適用します。 別のファイルを非表示にする/除外する場合、 **. vsls.json**ファイルは、これらの設定を使用してプロジェクトに追加できます。 参照してください[ファイルへのアクセスと可視性を制御する](../reference/security.md#controlling-file-access-and-visibility)詳細についてはします。

4. **コラボレーション セッションを開始します。**

    ここで、単に** をクリックして**「共有」のステータス バーの項目またはヒット**Ctrl + Shift + P/Cmd + Shift + P**を選択し、"ライブ共有。コラボレーション セッション (共有) を開始する"。

    ![共有ボタン](../media/vscode-share-button.png)

    > [!NOTE]
    > 初めてを共有するポートを開く Live 共有エージェントを許可するには、デスクトップのファイアウォール ソフトウェアによる求められること可能性があります。 これを受け入れるは完全に省略可能ですが、セキュリティで保護された「ダイレクト モード」を有効にしている場合に使用している人が同じネットワーク上がパフォーマンスを向上させる。 参照してください[接続モードを変更する](../reference/connectivity.md#changing-the-connection-mode)詳細についてはします。

    招待のリンクは、自動的にクリップボードにコピーされます。 ブラウザーで開くと、このリンクを使用するとこれらのフォルダーの内容を共有する新しいコラボレーション セッションに参加する他のユーザー。

    セッション状態を表す「共有」ステータス バー項目の遷移も表示されます。 参照してください[セッション状態](#session-states)次のように、以下の情報。

    招待のリンクを取得する必要がある場合は、もう一度、共有を開始した後セッション状態のステータス バーのアイコンをクリックしてもう一度アクセスを"(コピーのリンク) 他のユーザーを招待 を選択に注意してください。

5. **[省略可能]読み取り専用モードを有効にします。**

    コラボレーション セッションを開始するとは、ゲストが共有されているコードを編集したことを防ぐために読み取り専用にするセッションを設定できます。

    後の共有、招待のリンクをクリップボードにコピーされたことの通知が表示されます。 読み取り専用セッションを作成するオプションを選択することができます。

    ![VS コードの読み取り専用モード](../media/vscode-read-only-toast.png)

6. **リンクを送信する.**

    招待する場合に Skype などで Slack、電子メール経由でのリンクを送信します。 なおのレベルに、Live Share セッションは、ゲストに提供できるアクセス**のみを信頼できるユーザーと共有する必要があります**と共有している場合の影響を十分に検討します。

    > **セキュリティのヒント:** Live Share の機能の一部のセキュリティの影響について理解をしますか。 チェック アウト、[セキュリティ](../reference/security.md)記事。

    招待したゲスト、質問がある場合、"[クイック スタート。最初のセッションに参加](../quickstart/join.md)"記事の起動とゲストとして実行されているいくつか詳しく説明します。

7. **[省略可能]ゲストを承認します。**

    既定では、ゲストは、コラボレーション セッションを自動的に参加して、お客様と協力する準備ができているときに通知されます。 この通知には、セッションから削除するオプションが利用できますが、代わりに参加するすべてのユーザーに明示的な「承認」を要求することもできます。

    この機能を有効にするには、settings.json に、以下を追加だけです。

         "liveshare.guestApprovalRequired": true

    この設定を有効にした後は、通知からに参加させる前に、ゲストの承認を求められます。

    ![Visual Studio Code 参加の承認要求](../media/vscode-join-approval.png)

    参照してください[への招待および参加アクセス](../reference/security.md#invitations-and-join-access)招待セキュリティに関する考慮事項の詳細についてはします。

それです！！

2. **招待のリンクをクリックして/、お使いのブラウザーで招待状を開く**

    ここで、開くだけで (または再び開く)、ブラウザーでの招待のリンク。

    > **注**:Live Share の拡張機能をインストールしていない場合、拡張機能マーケットプ レースへのリンクが表示されます。 拡張機能のインストールし再起動、ツール、再試行してください。

    ブラウザーが Live Share を有効になっているツールを起動する必要があることに通知する必要があります。 選択したツールを起動する場合は、起動した場合は、共同作業セッションに接続できます。

    ![ページを参加します。](../media/join-page.png)

## <a name="share-a-server"></a>サーバーを共有します。

ゲストは正確な同じポートで、独自のローカル コンピューターからそのポートで共有するサーバーにアクセスすることになります。 たとえば、web サーバーを共有した場合**ポート 3000 で実行されている**でその同じ実行中の web サーバーにアクセスできます、ゲスト、**独自のマシン**で http://localhost:3000! これにより、ホストとゲスト間のセキュリティで保護された SSH または SSL トンネルを使用して実現は、共同作業セッションにのみアクセスを確認するために、サービスを使用して認証。

> **セキュリティのヒント:** ホストとして、ゲストで共有し、アプリケーションのポート (なくシステムのポートの共有) を引き続き使用するポートを備えた非常に選択的にあります。 ゲスト、ポート共有が、サーバー/サービスが独自のマシンで実行されている場合とまったく同じでは動作します。 これは非常に便利ですが、間違ったポートが共有されている場合があることができますも危険です。

セキュリティのために、指定したポートで実行されている唯一のサーバーは、他のゲスト使用できます。 さいわい、簡単に共同作業セッションとして 1 つの追加は**ホスト**します。 次の手順に従います。

1. VS Code アクティビティ バーまたはライブ共有エクスプ ローラー ビューで Live Share カスタム タブを開き、サーバーを選択して、"共有..."エントリまたはアイコンをクリックします。

    ![VS コード共有のローカル サーバー](../media/vscode-share-local-server-viewlet.png)<br />

1. サーバーが実行されているポート番号と、必要に応じて名前を入力します。

    ![ポート番号のプロンプトのスクリーン ショット](../media/vscode-enter-port.png)<br />

## <a name="share-a-terminal"></a>共有ターミナル

最新の開発では、さまざまなコマンド ライン ツールを頻繁に使用します。 さいわい、Live Share できます必要に応じて"ターミナル"と共有するゲストをホストとして。 共有ターミナルは読み取り専用または完全に共同作業ので、ゲストおよびできますコマンドを実行し、結果を参照してください。 ターミナル出力にゲストの可視性を提供またはテストの実践的なと実行、ビルド、またはコンピューターにのみ発生もトリアージ環境固有の問題を取得してそれらにできます。

ただし、端末が**いない**コマンドを実行する (ない場合はそれ自体のコマンドを実行する機能) の出力にゲスト少なくとも読み取り専用のアクセス権を付与するために、既定で共有します。 実際には、自由にローカル端末リスクなしでコマンドを実行し、のみ共有するときに、この方法は、これを行う必要があります。 さらに、ホストだけでは、1 つの起動となくを指定してください、監視を行ってからゲストを防ぐために共有端末を開始できます。

ホストとして、VS Code アクティビティ バーまたはライブ共有のエクスプ ローラー ビューで共有ライブのカスタム タブを開き、「共有サーバー...」を選択して、ターミナルを共有することができます。エントリまたはアイコンをクリックします。

![VS Code 共有ターミナル](../media/vscode-share-terminal-viewlet.png)<br />

この時点では、読み取り専用を選択したり、メニューからターミナルを読み取り/書き込みできます。 端末は、読み取り/書き込みが、すべてのユーザーを簡単に操作が好ましくないゲストを実行している場合に対処するホストを含む、ターミナルで入力できます。 ただし、安全にする必要があります**実際に必要なことがわかっている場合にのみをゲストに読み取り/書き込みアクセス権を付与**維持読み取り専用の端末にゲストをようにするシナリオを実行するコマンドの出力を参照してください。

> [!NOTE]
> 共同作業セッションが読み取り専用モードにある場合は、ホストによって読み取り専用の端末を共有できます。

![読み取り専用または読み取り/書き込みの選択](../media/vscode-share-terminal-ro-rw.png)<br />

開始する共有端末の種類を選択した場合は、新しい共有ターミナルは、VS Code の端末 タブで表示されます。

![共有ターミナルの実行](../media/vscode-share-terminal-up.png)<br />

複数端末は共有、または別のタブには、特定のターミナルに共有端末の一覧でエントリを選択してフォーカスが移動することができます。

![共有ターミナルにフォーカスします。](../media/vscode-shared-terminal-focus.png)<br />
